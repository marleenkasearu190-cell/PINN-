# PINN-

一个用于湖泊温度建模与数据处理的项目仓库，围绕 `ERA5`、`MODIS LST` 和 `PINN (Physics-Informed Neural Network)` 展开。当前仓库已经按版本整理为两个主要目录，便于区分早期脚本和后续整理版本。

## 仓库结构

```text
PINN-
├─ README.md
├─ .gitignore
├─ 第零版/
└─ 第一版/
```

### 第零版

[`第零版`](./第零版) 保存了较早放在仓库根目录的一组脚本，主要包括：

- `pinn.py`：PINN 建模主脚本
- `数据爬取.py`：数据下载相关脚本
- `提取.py`：数据提取脚本
- `物理参数.py`：物理参数与建模相关处理
- `原始下载数据可视化.py`：原始数据可视化
- `下载提取一体.py`：下载与提取整合脚本
- `运行Mendota流程.ps1` / `运行Mendota流程.bat`：流程运行脚本

这个目录适合作为早期整体流程版本留档。

### 第一版

[`第一版`](./第一版) 是进一步拆分后的版本，按功能划分为两个子目录：

- `下载爬取/`
- `数据处理/`

其中主要脚本包括：

- `下载爬取/ERA5下载转化一体.py`
- `下载爬取/lst下载.py`
- `下载爬取/ERA5-NC-数据爬取.py`
- `数据处理/可视化.py`
- `数据处理/预测模型.py`
- `数据处理/验证对比.py`

这个版本的结构更清晰，更适合作为后续继续整理和开发的基础。

## 项目用途

这个仓库主要用于以下任务：

- 下载 ERA5 湖泊相关气象与再分析数据
- 下载 MODIS 地表温度数据
- 对原始数据进行提取、转换与整理
- 绘制温度、深度、热力图等可视化结果
- 构建湖泊温度预测模型
- 使用 PINN 思路进行带物理约束的建模尝试
- 对模拟结果与观测结果进行验证对比

## 运行环境

建议使用：

```bash
Python 3.10+
```

常见依赖包括：

```bash
pip install pandas numpy matplotlib seaborn xarray netcdf4 scipy torch requests
```

如需运行部分下载脚本，还可能需要：

- `cdsapi`
- NASA Earthdata / AppEEARS 相关账号配置

## 使用建议

如果你是第一次阅读这个仓库，建议按下面顺序理解：

1. 先看 [`第一版`](./第一版) 的目录结构
2. 再看 [`第零版`](./第零版) 中的早期整合脚本
3. 优先从下载、提取、可视化、建模、验证这条主线理解整体流程
